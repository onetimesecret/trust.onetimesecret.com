# research/

Background and drafting material for policy pages we intend to add but have not
yet published. Held here so it stays out of the top-level document set (which is
reserved for the live legal documents) while remaining versioned and easy to
find when the corresponding page is drafted.

- `business-arrangements-we-support.md` — draft of the "Business arrangements
  we support" page. Per decision BA-1 (see `../REMAINING_DECISIONS.md`) this is
  the sole home of several commitments (sponsored arrangements, payer
  decoupling, consolidated invoicing, alias/boundary domains, spin-out path);
  retain or re-home those when the page moves toward publication. Do not publish
  the "promote domain to organization" capability claim until it is
  code-verified.
- `business-arrangements-rationale.md` — the Kinde-informed pricing/arrangement
  rationale behind the page above (formerly `ERRATA-1.md`).
- `research-approximated-architecture.md` — background on the Approximated
  edge/Custom-Domains architecture, for the eventual per-region data-residency
  and edge-TLS documentation.
